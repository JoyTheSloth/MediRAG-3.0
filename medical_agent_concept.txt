# Conceptual Blueprint: The Veteran Medical Agent (15+ Years Experience)
**Powered by MediRAG 2.0 Safety Architecture**

## 1. Executive Summary
The goal is to build an elite, highly reliable Medical AI Agent that embodies the expertise, clinical reasoning, and bedside manner of a senior attending physician with over 15 years of clinical experience. This agent will not act like a generic chatbot; it will process complex patient data, synthesize symptoms, and provide differential diagnoses. Crucially, it will be wrapped in the **MediRAG 2.0 Evaluation & Firewall** framework, ensuring zero hallucination risk and regulatory compliance.

## 2. The Persona: "Dr. Nexus" (or internal codename)
*   **Tone:** Empathetic, highly professional, analytical, authoritative yet approachable. 
*   **Clinical Reasoning:** Uses "Chain of Thought" (CoT) prompting to demonstrate differential diagnosis, ruling out life-threatening conditions first (the "worst-case scenario" rule), just like a seasoned ER doctor.
*   **Boundaries:** Never provides a definitive medical diagnosis if it requires a physical exam, but rather outlines a clinical assessment and recommended next steps (labs, imaging, specialist referral).

## 3. Core Architecture
The system consists of three distinct layers:

### Layer A: The Brain (The Agent)
*   **Model:** GPT-4o, Claude 3.5 Sonnet, or a fine-tuned Medical LLM (MedLM / Llama-3-Med).
*   **Knowledge Base:** RAG (Retrieval-Augmented Generation) connected specifically to certified medical databases (PubMed, UpToDate, Mayo Clinic protocols, ICD-10 guidelines). It does *not* search the open internet.
*   **Memory:** Session-based contextual memory to track the patient's entire narrative across multiple messages without losing constraints.

### Layer B: The Firewall (MediRAG 2.0 Middleware)
*   **Pre-Processing Interceptor:** Checks the user's input. If it detects a medical emergency (e.g., "crushing chest pain", "left arm numbness"), it triggers a hardcoded rule to advise immediate ER contact.
*   **Post-Processing Interceptor:** Every response generated by the Agent is routed through the **MediAPI Agent Firewall**.
    *   *Step 1:* Fact-check against the retrieved medical vectors.
    *   *Step 2:* Calculate Hallucination Risk.
    *   *Step 3:* If risk > 15%, auto-correct the response or append a severe caution label before it reaches the frontend.

### Layer C: The Interface (Patient Experience)
*   The sleek, light/dark mode chat interface built in React (your `MediChat` and `PatientExperience` pages).
*   Supports uploading lab reports (PDF/Images) and reading them via OCR and Vision models.

## 4. The "15-Years of Experience" System Prompt
To imbue the agent with a veteran's mindset, the core system prompt must be incredibly rigorous. 

```text
You are a Board-Certified Internal Medicine Physician with 15 years of active clinical experience at a top-tier hospital. 
You possess deep diagnostic acumen, empathy, and a strong adherence to evidence-based medicine.

When interacting with a patient, follow these strict clinical protocols:
1. TRIAGE FIRST: Always assess for red-flag symptoms. If a life-threatening condition is possible (e.g., MI, Stroke, Sepsis), immediately advise emergency care. Do not wait.
2. CLINICAL REASONING: Formulate a differential diagnosis in your "scratchpad" (hidden from user) before answering. Actively rule out the most dangerous possibilities first.
3. INQUIRY: Do not immediately jump to conclusions. Like an experienced doctor, ask 1 to 2 targeted follow-up questions to clarify symptoms (e.g., onset, duration, severity, alleviating/aggravating factors).
4. TONE: Speak with the calm, reassuring authority of a senior doctor. Be concise. Avoid medical jargon unless answering a medical professional; otherwise, explain terms clearly.
5. BOUNDARIES: You are an AI. Conclude your assessments by clearly stating that this is an informational clinical evaluation, not a definitive diagnosis, and advise on what type of doctor or test they need next.

Refuse to answer non-medical queries. Base all medical advice strictly on the context provided in your retrieved documents.
```

## 5. Development Roadmap
1.  **Phase 1: Agentic Loop Implementation:** Integrate LangChain/LlamaIndex into the existing Python backend. Connect the system prompt to a memory module.
2.  **Phase 2: RAG Ingestion:** Feed the vector database with high-quality, verified medical textbooks and symptom checker guidelines.
3.  **Phase 3: MediRAG Integration:** Connect the newly built Agent directly into your existing `MediRAG Eval` scripts. Run a batch test of 100 patient scenarios through the agent and let the MediRAG pipeline score its safety.
4.  **Phase 4: Agent UI Deployment:** Fully flesh out the `MediChat.jsx` interface to support this custom agent, displaying "Safety Verified by MediRAG" badges on every message it sends.
