1. INTRODUCTION
================================================================================

1.1 PURPOSE
-----------
This SRS is the authoritative reference for all MediRAG-Eval development,
testing, and evaluation activities. It is complete — a developer can implement
any module from this document alone without asking clarifying questions.

Demo scope (March 28): RAG backbone + Module 1 + RAGAS + Dashboard.
Full scope (May): All 5 modules + API + ablations + full report.

1.2 SCOPE
---------
MediRAG-Eval is a Python-based post-generation audit layer that:
- Receives : user query + retrieved context chunks + LLM-generated answer
- Produces : Hallucination Risk Score (0-100) + JSON audit report + Streamlit UI

It does NOT replace a medical QA system. It audits one.
It does NOT provide medical advice.

1.3 KEY DEFINITIONS
-------------------
RAG              Retrieval-Augmented Generation
Hallucination    Generated claim not supported by or contradicting retrieved context
Faithfulness     Degree to which every claim is entailed by retrieved context
NLI              Natural Language Inference (Entailment / Neutral / Contradiction)
NER              Named Entity Recognition
HRS              Hallucination Risk Score (0-100)
FAISS            Facebook AI Similarity Search
SciSpaCy         Allen AI biomedical NER library
RAGAS            Retrieval Augmented Generation Assessment
DeBERTa          cross-encoder/nli-deberta-v3-base
BioBERT          dmis-lab/biobert-v1.1
Atomic Claim     Single indivisible verifiable assertion from a sentence
Chunk Metadata   Structured fields stored alongside each FAISS vector (FR-03b)
RxCUI            RxNorm Concept Unique Identifier — canonical drug ID
Cold Start       First inference call after process launch (model loading delay)

--- RAG BACKBONE ---

FR-01: DOCUMENT INGESTION
Ingest and index from:
- PubMed Open Access Corpus (FTP — free)
- BioASQ 2023 (bioasq.org — free registration)
- MedQA-USMLE (GitHub: jind11/MedQA — free)
- PubMedQA (HuggingFace: pubmed_qa — free)

FR-02: DOCUMENT CHUNKING
- Chunk size: 512 tokens, overlap: 50 tokens
- Chunker: LangChain RecursiveCharacterTextSplitter

FR-03: EMBEDDING GENERATION
- Model: dmis-lab/biobert-v1.1 (768-dim dense vectors)
- Same model for documents and queries

FR-03b: CHUNK METADATA SCHEMA
Parallel dict alongside FAISS index, saved as pickle:
  metadata_store: dict[int, dict]   # key = FAISS integer index

  {
    "chunk_id":     str,   # uuid4
    "doc_id":       str,   # PubMed PMID or dataset row ID
    "source":       str,   # "pubmed"|"bioasq"|"medqa"|"pubmedqa"
    "title":        str,
    "pub_type":     str,   # see FR-11b
    "pub_year":     int,   # 0 if unknown
    "journal":      str,   # "" if unknown
    "chunk_index":  int,
    "total_chunks": int
  }

pub_type extraction:
  PubMed:   <PublicationType> from XML
  BioASQ:   "type" field from JSON metadata
  MedQA:    hardcode "exam_question"
  PubMedQA: hardcode "research_abstract"

FR-04: VECTOR RETRIEVAL
- Default top-k: 5 (config: retrieval.top_k)
- FAISS IndexFlatIP with L2-normalized vectors (= cosine similarity)
- Returns: list of (chunk_text: str, metadata: dict, score: float)

--- MODULE 1: FAITHFULNESS SCORER ---

FR-05: CLAIM DECOMPOSITION
Model: Mistral-7B via Ollama, temperature: judge_temperature (0.0)

EXACT PROMPT TEMPLATE:
----PROMPT START----
You are a medical claim extractor. Decompose the following medical answer into
atomic, verifiable claims. Each claim must:
1. Be a single sentence with exactly one verifiable assertion
2. Be self-contained
3. Not include hedging phrases unless clinically significant
4. Not duplicate another claim

Return ONLY a JSON array of strings. No explanation, no markdown, no preamble.
Example: ["Metformin treats Type 2 Diabetes.",
          "Standard starting dose of Metformin is 500mg twice daily.",
          "Metformin is contraindicated in patients with eGFR below 30."]

Medical Answer:
{answer}
----PROMPT END----

Parse: strip markdown backticks → json.loads()
Fallback on parse failure: answer.split(". ")
Log parse failures at WARNING.

FR-06: NLI ENTAILMENT CHECK
Model: cross-encoder/nli-deberta-v3-base, batch_size from config

Token truncation (512 limit):
  claim_ids = tok(claim, add_special_tokens=False)["input_ids"]
  max_ctx   = 510 - len(claim_ids)
  ctx_ids   = tok(context, add_special_tokens=False)["input_ids"][-max_ctx:]

Classification:
  ENTAILED     — max entailment score across all chunks >= 0.75
  CONTRADICTED — max contradiction score >= 0.75
  NEUTRAL      — neither threshold met

On CUDA OOM: retry with batch_size=1, log WARNING.

FR-07: FAITHFULNESS SCORE
  Faithfulness = |ENTAILED| / |total claims|
  Empty answer → 0.0, log WARNING.

--- MODULE 2: MEDICAL ENTITY VERIFIER ---

FR-08: ENTITY EXTRACTION
Model: SciSpaCy en_core_sci_lg
Types: DRUG (T121/T195/T200), DOSAGE (digit+unit regex), CONDITION (T047/T184)
Dosage regex: r'\b\d+\.?\d*\s*(mg|mcg|ml|g|IU|units|mmol|mEq)\b'

FR-09: ENTITY MISMATCH DETECTION
Drug normalization:
  1. RxNorm REST (3s timeout): rxnav.nlm.nih.gov/REST/rxcui.json?name={name}&search=1
  2. On failure: lookup in data/rxnorm_cache.csv (exact then lowercase)
  3. On cache miss: use raw drug_name, log WARNING

Severity (apply in priority order):
  CRITICAL:
    a) DOSAGE numerical value differs > 10% from context
    b) DRUG not found in any retrieved chunk after normalization
    c) DRUG appears in context as "contraindicated"/"avoid"/"not for use in"
       but answer recommends it
  MODERATE:
    a) DRUG in different semantic role in context vs answer
    b) CONDITION not in any chunk
  MINOR: terminology/abbreviation mismatches

Dosage tolerance: abs(answer - context) / context > 0.10 → CRITICAL
Incompatible units → MODERATE, log DEBUG.

FR-10: ENTITY ACCURACY SCORE
  Entity Accuracy = |verified| / |total extracted|

--- MODULE 3: SOURCE CREDIBILITY RANKER ---

FR-11a: KEYWORD-BASED (demo)
  Tier 1: "randomized controlled trial","randomised controlled trial","rct",
          "systematic review","meta-analysis","cochrane"
  Tier 2: "clinical guideline","who guideline","nih guideline","icmr",
          "practice guideline","consensus statement"
  Tier 3: "cohort study","observational study","prospective study",
          "retrospective study","cross-sectional"
  Tier 4: "case report","case study","case series"
  Default: Tier 5

FR-11b: METADATA-BASED (May)
  "randomized-controlled-trial" → 1 | "systematic-review" → 1
  "meta-analysis" → 1           | "practice-guideline" → 2
  "consensus-development" → 2   | "journal-article" → 3
  "observational-study" → 3     | "case-reports" → 4
  "letter" → 4                  | "editorial" → 4
  "exam_question" → 5           | "research_abstract" → 3
  anything else → 5

FR-12: SOURCE CREDIBILITY SCORE
  tier_weights = {1:1.00, 2:0.85, 3:0.65, 4:0.40, 5:0.20}
  Source Credibility = mean(tier_weights[tier] for each chunk)

--- MODULE 4: CONTRADICTION DETECTOR ---

FR-13: CROSS-SENTENCE CONSISTENCY
Model: cross-encoder/nli-deberta-v3-base (reuse loaded instance from Module 1)
Apply same 512-token truncation as FR-06.
N > 10 sentences: check adjacent pairs + (first, last) only.
Flag pairs with CONTRADICTION confidence > 0.75.

FR-14: CONTRADICTION RISK SCORE
  Contradiction Risk = |contradicted pairs| / |total checked pairs|

--- RAGAS INTEGRATION ---

FR-15: RAGAS METRICS
LLM judge: Mistral-7B via Ollama (temperature: 0.0)
Embeddings: BioBERT via HuggingFaceEmbeddings (NOT OllamaEmbeddings)

Integration:
  from langchain_community.chat_models import ChatOllama
  from langchain_community.embeddings import HuggingFaceEmbeddings
  from ragas import evaluate
  from ragas.metrics import (faithfulness, answer_relevancy,
                              context_precision, context_recall)
  from ragas.llms import LangchainLLMWrapper
  from ragas.embeddings import LangchainEmbeddingsWrapper

  llm = ChatOllama(model="mistral", base_url="http://localhost:11434",
                   temperature=0.0)
  ragas_llm = LangchainLLMWrapper(llm)

  hf_emb = HuggingFaceEmbeddings(
      model_name="dmis-lab/biobert-v1.1",
      model_kwargs={"device": "cpu"},
      encode_kwargs={"normalize_embeddings": True}
  )
  ragas_emb = LangchainEmbeddingsWrapper(hf_emb)

  for metric in [faithfulness, answer_relevancy, context_precision,
                 context_recall]:
      metric.llm = ragas_llm
  answer_relevancy.embeddings = ragas_emb

context_recall ground_truth:
  PubMedQA: row["long_answer"]
  MedQA:    row["answer"] (correct option text)
  BioASQ:   row["ideal_answer"]
  Unknown:  skip context_recall, report as null, log WARNING
  NEVER pass "" as ground_truth — causes division by zero.

  metrics = [faithfulness, answer_relevancy, context_precision]
  if ground_truth:
      metrics.append(context_recall)

--- SCORE AGGREGATION ---

FR-16: HALLUCINATION RISK SCORE
  weighted = (0.40 × Faithfulness) + (0.25 × EntityAccuracy)
           + (0.20 × SourceCredibility) + (0.15 × (1 - ContradictionRisk))
  HRS = round(100 × (1 - weighted))
  Bands: 0-30 LOW, 31-60 MODERATE, 61-85 HIGH, 86-100 CRITICAL

FR-20: build_rxnorm_cache.py [NEW — fixes Gap #1]
------------------------------------------------------
Full specification for the offline drug normalization cache builder.

PURPOSE:
  Read DrugBank Open Data XML → extract all drug names and synonyms →
  query RxNorm REST API for each → save to data/rxnorm_cache.csv

INPUT:
  --drugbank-xml : path to DrugBank Open Data XML file
                   (default: data/raw/drugbank_open_data.xml)
  --output-csv   : path for output CSV
                   (default: data/rxnorm_cache.csv)
  --delay        : seconds between API calls (default: 0.1)
                   RxNorm is free but rate-limit courtesy applies

OUTPUT CSV SCHEMA:
  drug_name      : str  — drug name or synonym as appears in DrugBank XML
  rxcui          : str  — RxNorm concept ID ("" if not found)
  canonical_name : str  — RxNorm preferred name ("" if not found)

IMPLEMENTATION SPEC:

  import xml.etree.ElementTree as ET
  import requests, csv, time, argparse, logging

  RXNORM_URL = "https://rxnav.nlm.nih.gov/REST/rxcui.json"
  NS = {"db": "http://www.drugbank.ca"}   # DrugBank XML namespace

  def extract_drug_names(xml_path: str) -> list[str]:
      """Extract all drug names and synonyms from DrugBank Open Data XML."""
      tree = ET.parse(xml_path)
      root = tree.getroot()
      names = set()
      for drug in root.findall("db:drug", NS):
          # Primary name
          name_el = drug.find("db:name", NS)
          if name_el is not None and name_el.text:
              names.add(name_el.text.strip())
          # Synonyms
          for syn in drug.findall("db:synonyms/db:synonym", NS):
              if syn.text:
                  names.add(syn.text.strip())
          # International brands
          for brand in drug.findall("db:international-brands/db:international-brand/db:name", NS):
              if brand.text:
                  names.add(brand.text.strip())
      return sorted(names)

  # FIX v1.4: use /approximateTerm — returns rxcui AND name in ONE call.
  # Previous 2-call approach (rxcui lookup + name lookup) doubled API traffic.
  RXNORM_APPROX_URL = "https://rxnav.nlm.nih.gov/REST/approximateTerm.json"

  def query_rxnorm(drug_name: str, timeout: int = 5) -> tuple[str, str]:
      """Returns (rxcui, canonical_name). Returns ("", "") on failure.
      Uses /approximateTerm — single HTTP call (option=1 = normalize spelling)."""
      try:
          r = requests.get(RXNORM_APPROX_URL,
                           params={"term": drug_name, "maxEntries": "1",
                                   "option": "1"},
                           timeout=timeout)
          if r.status_code != 200:
              return "", ""
          candidates = (r.json()
                        .get("approximateGroup", {})
                        .get("candidate", []))
          if not candidates:
              return "", ""
          rxcui = candidates[0].get("rxcui", "")
          name  = candidates[0].get("name", drug_name)  # fallback to input
          return rxcui, name
      except Exception:
          return "", ""

  def main():
      parser = argparse.ArgumentParser()
      parser.add_argument("--drugbank-xml", default="data/raw/drugbank_open_data.xml")
      parser.add_argument("--output-csv",   default="data/rxnorm_cache.csv")
      parser.add_argument("--delay",        type=float, default=0.1)
      args = parser.parse_args()

      names = extract_drug_names(args.drugbank_xml)
      logging.info(f"Extracted {len(names)} drug names from DrugBank XML")

      with open(args.output_csv, "w", newline="", encoding="utf-8") as f:
          writer = csv.writer(f)
          writer.writerow(["drug_name", "rxcui", "canonical_name"])
          for i, name in enumerate(names):
              rxcui, canonical = query_rxnorm(name)
              writer.writerow([name, rxcui, canonical])
              if i % 100 == 0:
                  logging.info(f"Progress: {i}/{len(names)}")
              time.sleep(args.delay)

      logging.info(f"Cache saved to {args.output_csv}")

  if __name__ == "__main__":
      logging.basicConfig(level=logging.INFO)
      main()

RUNTIME ESTIMATE:
  ~14,000 drug names × 0.1s delay × 1 API call each ≈ ~24 minutes  [FIXED v1.4]
  Can be run during the day before demo (not just overnight).
  Do NOT run during active demo prep.

RETRY LOGIC:
  query_rxnorm() returns ("", "") on any failure — no retry needed.
  The entity_verifier will handle cache misses gracefully (FR-09).

USAGE:
  python scripts/build_rxnorm_cache.py
  python scripts/build_rxnorm_cache.py --drugbank-xml /path/to/file.xml
  python scripts/build_rxnorm_cache.py --delay 0.2   # slower, more polite

16. MODEL WARMUP STRATEGY [NEW — fixes Gap #3]
================================================================================

PURPOSE:
  On first inference call, each model has a cold-start delay:
    Mistral-7B (Ollama) : 10-30 seconds to load into VRAM/RAM
    BioBERT embeddings  : 5-10 seconds to load into memory
    DeBERTa-v3          : 5-10 seconds to load into memory
    SciSpaCy en_sci_lg  : 3-5 seconds

  During a live 5-minute demo, a 30-second cold-start on the first query is
  unacceptable. Run warmup.py BEFORE opening Streamlit on stage.

scripts/warmup.py FULL SPEC:

  import requests, time, logging
  from transformers import pipeline as hf_pipeline, AutoTokenizer
  from langchain_community.embeddings import HuggingFaceEmbeddings
  from langchain_community.chat_models import ChatOllama
  import spacy, yaml

  logging.basicConfig(level=logging.INFO,
                      format="%(asctime)s [%(levelname)s] %(message)s")
  logger = logging.getLogger("warmup")

  config = yaml.safe_load(open("config.yaml"))

  def warmup_ollama():
      logger.info("Warming up Ollama / Mistral...")
      llm = ChatOllama(
          model=config["llm"]["model"],
          base_url=config["llm"]["base_url"],
          temperature=0.0
      )
      response = llm.invoke("Respond with the single word: READY")
      logger.info(f"Ollama warm — response: {response.content.strip()}")

  def warmup_biobert():
      logger.info("Warming up BioBERT embeddings...")
      emb = HuggingFaceEmbeddings(
          model_name=config["retrieval"]["embedding_model"],
          model_kwargs={"device": "cpu"},
          encode_kwargs={"normalize_embeddings": True}
      )
      _ = emb.embed_query("warmup query")
      logger.info("BioBERT warm.")

  def warmup_deberta():
      """IMPORTANT: This creates a THROW-AWAY DeBERTa instance for verification.
      It proves the model downloads and loads correctly. This instance is NOT
      reused — it is garbage-collected after this function returns.
      The main pipeline (src/evaluate.py) creates its own SHARED singleton
      that is passed to both FaithfulnessScorer and ContradictionDetector.
      Two separate loads are intentional: warmup verifies, main provides shared."""
      logger.info("Warming up DeBERTa NLI (verification load)...")
      pipe = hf_pipeline(
          "text-classification",
          model=config["modules"]["faithfulness"]["nli_model"],
          device=-1,
          batch_size=1
      )
      _ = pipe("Test claim. [SEP] Test context.")
      logger.info("DeBERTa warm (verification instance discarded).")

  def warmup_scispacy():
      logger.info("Warming up SciSpaCy...")
      nlp = spacy.load(config["modules"]["entity_verifier"]["spacy_model"])
      _ = nlp("Metformin 500mg for Type 2 Diabetes.")
      logger.info("SciSpaCy warm.")

  def warmup_faiss():
      logger.info("Verifying FAISS index...")
      import faiss, pickle
      idx  = faiss.read_index(config["retrieval"]["index_path"])
      meta = pickle.load(open(config["retrieval"]["metadata_path"], "rb"))
      logger.info(f"FAISS index has {idx.ntotal} vectors, "
                  f"{len(meta)} metadata entries.")

  def warmup_save_fallback():
      """Run a real evaluation on Q1 and save to demo/demo_fallback.json.
      Only saves if the file does not already exist (idempotent)."""
      import os, json, sys
      os.makedirs("demo", exist_ok=True)
      fallback_path = "demo/demo_fallback.json"
      if os.path.exists(fallback_path):
          logger.info(f"Fallback JSON already exists at {fallback_path} — skipping.")
          return
      logger.info("Generating demo_fallback.json (may take ~30s)...")
      try:
          sys.path.insert(0, "src")
          from evaluate import run_evaluation  # src/evaluate.py
          query = ("What is the recommended dosage of Metformin for "
                   "Type 2 Diabetes in elderly patients with kidney disease?")
          result = run_evaluation(query)  # returns dict matching FR-17
          with open(fallback_path, "w", encoding="utf-8") as f:
              json.dump(result, f, indent=2, ensure_ascii=False)
          logger.info(f"Fallback JSON saved to {fallback_path}")
      except Exception as e:
          logger.error(f"Could not generate fallback JSON: {e}")
          logger.warning("Create demo/demo_fallback.json manually before demo day.")

  if __name__ == "__main__":
      start = time.time()
      warmup_faiss()
      warmup_biobert()
      warmup_deberta()
      warmup_scispacy()
      warmup_ollama()
      warmup_save_fallback()   # NEW v1.4 — auto-generates demo/demo_fallback.json
      elapsed = time.time() - start
      logger.info(f"All models warm. Total warmup time: {elapsed:.1f}s")
      logger.info("Safe to launch Streamlit dashboard.")

USAGE:
  python scripts/warmup.py
  # Wait for "All models warm" message
  # THEN launch: streamlit run src/dashboard/app.py

pytest.ini (place in project root):
  [pytest]
  testpaths = tests
  python_files = test_*.py
  python_classes = Test*
  python_functions = test_*
  addopts = -v --tb=short

  because Python path is not set. The ini ensures tests/ is discovered and
  src/ modules are importable.

    # conftest.py
    import sys, os
    sys.path.insert(0, os.path.join(os.path.dirname(__file__), "src"))
  This ensures `import src.modules.faithfulness` works in all test files.

.gitattributes (place in project root):
  # Normalize all text files to LF on commit (prevents CRLF corruption)
  * text=auto

  # Explicit LF for all source and config files
  *.py   text eol=lf
  *.txt  text eol=lf
  *.md   text eol=lf
  *.yaml text eol=lf
